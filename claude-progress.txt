ARCHITECTURE HIGHLIGHTS
================================================================================

This codebase follows the CRITICAL dual build target architecture:

1. **ONE SOURCE TREE, TWO OUTPUTS**
   - H5: npm run dev:h5 → localhost:5173 → Puppeteer verification
   - mp-weixin: npm run build:mp-weixin → WeChat Mini Program

2. **PLATFORM ADAPTER ISOLATION**
   - ALL WeChat APIs (wx.*, uni.cloud.*) are isolated in src/platform/
   - Components/stores NEVER call wx.* directly
   - Conditional compilation (#ifdef MP-WEIXIN / #ifdef H5)
   - H5 mocks are fully functional (not stubs)

3. **VERIFICATION PROTOCOL**
   - Every feature needs screenshot OR test output evidence
   - Evidence must be Read before updating feature_list.json
   - verify-gate hook enforces this rule

4. **NO WEB-ONLY DEPENDENCIES**
   - ✅ uView Plus (uni-app compatible)

   - ❌ NO Vant, Element Plus, echarts (DOM-only)
